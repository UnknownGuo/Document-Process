# 自动化文档处理微创业项目 (AutoDoc-Agent) - 终极技术方案 V2.1

## 一、 核心架构理念：自然语言驱动 + 工业级闭环质检
本项目致力于打破传统自动化文档处理中“格式易丢失、代码难维护、生成质量差”的死穴。通过引入**“大模型大脑（Multi-Agent）”**与**“无损注入手（Surgical Injector）”**的解耦架构，实现所见即所得的文档批量生产。

---

## 二、 完整业务流转标准操作程序 (SOP)

### 阶段 1：业务模板配置（用户侧 - Windows）
1.  **直觉化占位**：在 Word 模板中，凡需 AI 填写处，直接打上中括号指令 `【这里写具体要求】`。
2.  **样式预设锁定**：用户可选择性地对 `【】` 及其内容设置字号、中文字体（如黑体）、加粗、颜色及对齐方式。系统遵循“有所见即有所得”原则，用户设置的格式越细，继承越准；未设置的按默认格式。
3.  **无损继承原则**：系统底层引擎采用纯字符串安全替换技术，100% 继承上述样式。严禁使用破坏 XML 结构的暴力删改，彻底杜绝“文档无法读取”报错。

### 阶段 2：意图对齐与单篇验证（双端协作）
1.  **初稿试跑**：系统提取指令并生成 V1 版样稿。
2.  **人工意图反馈 (Modify Loop)**：
    *   用户在样稿上进行微调（建议开启“修订模式”）。
    *   系统自动扫描 `w:ins` 和 `w:del` 标签，深度透视用户的隐性意图（如：“标题不加前缀”、“星星必须分行”）。
3.  **约束固化**：系统向用户汇报理解到的“意图法则”，确认后自动存入该 Case 的 `Constraints` 质检库。

### 阶段 3：混合智能管线生成（系统侧 - Linux/WSL）
核心引擎采用 **Writer + Reviewer** 双智能体架构：
1.  **数据流扁平化准则（V2.1 新增）**：
    *   **严格禁止**在生成链路中传递包含 `context` 或 `metadata` 的嵌套 JSON 对象。
    *   **强制规定**：AI 返回的 JSON 必须为纯净的 `[Block_ID] : [纯中文内容]` 键值对。
    *   **目标**：彻底切断元数据（如 `contextcurrent_text`）泄露至最终 Word 文档的逻辑路径。
2.  **写作者 (Writer)**：负责起草内容。
3.  **质检员 (Reviewer)**：拿着阶段 2 固化的规则，对内容进行 1-10 分的严格审核。**若不达标则强制打回重写，死磕质量（上限可配置为 5 次或直至 8 分以上为止）**。

### 阶段 4：高阶质量控制管线 (Advanced QA Pipeline)
1.  **专业校对 (Proofreader Agent)**：由“特级教研员”角色对通过质检的内容进行错别字查杀、语病修复和话术润色。
2.  **底层 XML 格式扫描 (XML Deep Diff)**：在保存前，自动比对成品与模板的 XML 字体节点属性。若发现格式降级（如变为 Calibri），系统立即隔离并报警。
3.  **多模态视觉一致性校验 (Visual Consistency Check)**：利用 Gemini Pro Vision 对比模板与成品截图，确保段落密度美观、无排版畸变。
4.  **持续强化学习 (Continuous Learning)**：维护 `history_intent.json`。AI 每次执行前强制预读“历史纠错集”，实现“同样的错绝不犯第二次”。

### 阶段 5：自动化批量交付
*   **归档路径**：`A:\AI_Studio\Auto-docs\xxx\output\`
*   **命名规范**：`课程名_v[n]_FinalQA_[YYYYMMDD].docx`

## 三、 技术栈与防御策略
- **大脑**：Gemini 2.5 Pro (1.5 Pro 备选)。
- **工具**：`python-docx` (读取/安全注入), `lxml` (底层 XML 透视), `Pillow` (视觉处理)。
- **防御**：
    *   **反幻觉**：Reviewer 强制打回机制。
    *   **反损坏**：禁止删除 Paragraph/Table 节点，仅执行 Run 内容替换。
    *   **反泄露**：`.env` 环境变量隔离 API Key。
